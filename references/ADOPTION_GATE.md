# Adoption Gate

Use this gate to prevent the kit from turning into a feature blob.

A new idea, open-source project, paper, or workflow can enter the core only if it satisfies all criteria:

1. It strengthens K0-K6.
2. It can be implemented as a command, skill, hook, template, script, or measurable check.
3. It has a clear metric: fewer reopens, fewer forbidden file violations, higher first-pass harness rate, better handoff success, lower context cost, or fewer drift incidents.
4. It does not require users to maintain a new long-lived document by hand unless the benefit is clear.
5. It is default-off unless it is part of the kernel.

No new noun without new executable behavior.

Research states:

- Watch: interesting, no integration.
- Adapt: optional skill or template candidate.
- Adopt: enters default workflow.

Every Adopt item must also define a kill criterion.
